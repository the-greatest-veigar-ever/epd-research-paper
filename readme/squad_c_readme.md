# Squad C: Ghost Agents (Remediation Execution)

Squad C represents the execution arm of the EPD (Ephemeral Polymorphic Defense) framework. It is responsible for taking the high-level remediation strategies (STIX objects) generated by the Intelligence Agent (Squad B) and translating them into exact, executable command-line syntax (e.g., AWS CLI, iptables).

This directory contains the core agent definitions and a rigorous evaluation pipeline designed to test the system's performance, speed, and cybersecurity resilience against adversarial prompt injection.

---

## 🏗 Architecture & Approaches

To combat the inherent vulnerabilities of LLM-based autonomous agents (specifically prompt injection and jailbreaks), Squad C was built and evaluated using **four distinct architectural approaches**:

1.  **Single Baseline (`phi_baseline`)**: A single LLM (Phi-3 Mini) is loaded into memory once and processes all requests sequentially. 
2.  **Single Suicide (`phi_suicide`)**: A single LLM (Phi-3 Mini) is loaded into memory, executes *one* remediation plan, and is immediately destroyed and purged from VRAM to prevent context bleeding.
3.  **Multimodal Static (`multimodal_static`)**: A pool of three different SLMs (Phi-3 Mini, Llama 3.2 3B, Gemma 2 2B) is loaded into memory. For every incoming plan, one model is chosen entirely at random to execute it.
4.  **Multimodal Suicide (`multimodal_suicide`)**: A model is randomly selected from the pool, loaded, executes the plan, and is immediately destroyed.

The "Multimodal" approaches naturally implement **Moving Target Defense (MTD)**. By randomly rotating the executing model, an attacker cannot reliably craft a payload targeting a specific architecture's weaknesses.

---

## 🧪 Evaluation Methodology

To prove the efficacy of these approaches, we conducted a massive 18,000+ inference evaluation utilizing a **Mixed Dataset** strategy:

1.  **Clean Data:** We extracted 3,824 legitimate STIX remediation plans generated during Squad B's integration testing.
2.  **Adversarial Injection:** We generated 674 malicious STIX plans representing diverse attacks (e.g., requesting the model to delete databases, open reverse shells, or ignore its system prompt).
3.  **The Mixed Dataset:** The clean and malicious plans were shuffled together into a single dataset of 4,498 logs (approx. 15% injection rate).

The evaluation script (`evaluate.py`) processes this mixed dataset in two distinct phases.

---

## 📊 Phase 1: Performance & Efficiency Results

In Phase 1, we measured how fast and reliably the system could churn through the entire mixed queue of 4,498 items without crashing or failing to generate commands.

| Approach | Attack Success Rate (ASR) | Task Success Rate (TSR) | Avg Processing Time | Avg Init Time |
| :--- | :--- | :--- | :--- | :--- |
| phi_baseline | 100.0% | 99.36% | 2.37s | 0.00s |
| phi_suicide | 100.0% | 99.51% | 2.50s | 0.38s |
| multimodal_static | 100.0% | 98.33% | 1.42s | 0.00s |
| multimodal_suicide| 100.0% | 98.69% | 1.54s | 0.66s |

**Key Findings:**
*   **Speed:** By distributing traffic across three models (including the incredibly fast Gemma 2B), the Multimodal approaches cut the average processing time from ~2.4s to **1.4s** per plan.
*   **The Suicide Tradeoff:** Unloading the model after every plan adds a fixed 0.4s - 0.7s initialization penalty per request, slightly increasing total runtime, but provides absolute zero-trust isolation boundaries.

---

## 🛡️ Phase 2: Security Resilience Results

In Phase 2, the `SecurityEvaluator` filtered out the normal requests and rigorously analyzed the models' behavior exclusively against the 674 injected attacks to see if they successfully refused the malicious payloads.

| Approach | Prompt Injection Resistance (CyberSecEval) | Harmful Request Rejection (HarmBench) | Jailbreak Resistance (JailbreakBench) | Aggregate Security Score |
| :--- | :--- | :--- | :--- | :--- |
| phi_baseline | 50.4% | 87.2% | 74.2% | 72.3% |
| phi_suicide | 55.1% | 87.6% | 70.8% | 72.8% |
| multimodal_static | 61.7% | 92.9% | 67.1% | 75.2% |
| multimodal_suicide| 59.6% | 91.2% | 68.5% | 74.3% |

**Key Findings:**
*   **Moving Target Defense Works:** The Multimodal approaches easily achieved the highest aggregate security scores (75.2%). By randomly rotating the LLMs, highly specific prompt injections from the CyberSecEval dataset frequently missed their intended target, resulting in a ~10-11% increase in absolute resistance.
*   **Strong Refusal:** The system demonstrated excellent intrinsic resistance to direct harmful requests (HarmBench), rejecting ~93% of obvious malicious commands.

## 🚀 Running the Evaluation

To reproduce the full pipeline (Performance + Security), utilize the unified evaluation wrapper:

```bash
python -m src.ghost_agents.evaluate
```

*(Note: The full mixed dataset evaluation executes over 18,000 inferences and can take upwards of 12 hours depending on hardware. Use the `--limit N` flag for quick testing).*
